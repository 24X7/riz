Recurring semantic pass over what the lints cannot see (docs/SAFETY.md,
enforcement tier "review"), plus the ratchet burn-down. One iteration per
module group; counts from `scripts/safety-check.sh` (production code,
`--lib --bins`, unique violation sites).

### Behavior changes (all covered by new tests)

1. Empty `choices` from the LLM gateway: task fails with a status message
   instead of panicking the task's tokio task (`a2a.rs`).
2. `POST /_riz/connections/{id}` on a backlogged connection: **429** instead
   of unbounded buffering (`ws/management.rs`;
   `post_returns_429_when_outbound_queue_full`).
3. A2A task store at capacity: oldest terminal task evicted, or the new task
   rejected with −32603 when all 1024 retained tasks are live (`a2a.rs`;
   `task_store_rejects_insert_when_full_of_live_tasks`,
   `task_store_evicts_oldest_terminal_task_at_cap`).
4. Pathological timeout configs clamp instead of panicking
   (`deadline_after_saturates_instead_of_panicking`).

Everything else is behavior-preserving: the `serde_json` `[]` → `.get()`
conversions are observationally identical (`Value`'s `Index` returns Null for
missing keys), and the function splits move code without changing it.

### Behavior changes (all covered by new tests)

1. An `Origin` header containing an ASCII control character (e.g. NUL):
   CORS headers are omitted (treat-as-absent) instead of panicking the
   request task (`preflight_origin_with_control_char_treated_as_absent`,
   `response_headers_origin_with_control_char_treated_as_absent`).
2. Inbound WebSocket frames over 32 KiB (or messages over 128 KiB) close
   the connection without dispatching, matching AWS quotas
   (`websocket_oversized_inbound_frame_closes_connection` — also asserts a
   normal echo works on the same connection first).
3. `percent_decode("%4")` returns `"%4"` instead of `"%"` — truncated
   sequences no longer drop the consumed byte
   (`percent_decode_truncated_sequence_keeps_consumed_bytes`).
4. Everything else is behavior-preserving and now pinned: config-list
   fallbacks cannot inject headers (`preflight_malformed_allow_methods_*`,
   `*_allow_headers_*`, `response_malformed_expose_headers_*`),
   `max_age_secs = u64::MAX` renders exactly
   (`preflight_max_age_renders_u64_decimal`), and the auth cache key
   domain-separates absent-vs-empty credentials
   (`auth_cache_key_none_differs_from_empty_header`). The middleware split

### Behavior changes (all covered by new tests)

1. **Crash-looping workers now trip the circuit breaker**: five consecutive
   worker deaths with no successful response in between mark the pool
   unhealthy (503 / fail-closed guards) instead of respawn-per-request
   forever (`crash_loop_with_successful_respawns_trips_the_breaker`).
2. **Watcher respawns back off**: 200 ms doubling to a 5 s ceiling under
   consecutive crashes; first-death respawn latency is unchanged
   (`respawn_backoff_is_monotone_and_capped`).
3. **Cancelled generic invocations kill the worker** (previously only the
   HTTP path did): no stale response line can answer a later WS/guard
   invocation (`dropped_stdio_roundtrip_kills_the_worker`).
4. **Failed pipe wiring is a spawn error, not a panic**
   (`wire_stdio_transport_without_pipes_is_an_error_not_a_panic`), and a
   transport-dispatch bug fails one invocation, not the daemon
   (`stdio_roundtrip_on_runtime_api_worker_is_an_error_not_a_panic`).
5. **Worker output is bounded**: log lines cap at 8 KiB (truncated +
   marked — `tail_to_logs_caps_runaway_lines`), stdio response lines at
   16 MiB (over-cap → malformed-response kill+respawn), and hostile wasm
   ABI lengths fault before allocating
   (`guest_range_rejects_hostile_pairs_before_allocating`).

Everything else is behavior-preserving: the invoke restructure moves the
existing arms behind shared helpers without changing any outcome mapping
(pinned by the full pool/WS/guard integration suites), the `.get()`
conversions in `runtime/mod.rs` are pinned by the 16 pre-existing
route-matching tests, and `GuardDecision` encodes the exact allow/deny/
fail-closed semantics the string match implemented (pinned by the existing
guard e2e tests).
