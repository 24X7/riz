# Shape-purity Tranche 4 (PR5 — broker into the daemon) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The capability broker moves from each `__wasm-host` child into the daemon behind a UDS + per-worker-token boundary, backed by a shared `sqlx::PgPool` per resource — making R2's three structural promises true: shared pool, per-function-correct limits, credentials in exactly one process.

**Architecture:** New `src/broker/{wire,server,client}.rs`. The daemon starts a `BrokerService` (supervised accept loop mirroring `TelemetrySupervisor`, `src/observability/mod.rs:134-398`): 0700 tmpdir + 0600 socket, HELLO(token) auth mapped token→function, per-connection cap, CALL/REPLY strict request-response (BCP frames: `u32 len | u8 ver | u8 type | u16 flags | u64 call_id | payload`; CALL payload `u16 verb_len | verb | u16 grant_len | grant | body`). The child keeps NO tokio: `arm_broker`/`HostCtx.{broker,rt}`/`block_on` (wasm.rs:103-113,182-202,378-391) are replaced by a sync `CapabilityClient` (`std::os::unix::net::UnixStream`, SO_RCVTIMEO/SNDTIMEO = max grant `call_timeout_ms` + 250ms via `RIZ_BROKER_TIMEOUT_MS`), lazy-connect, reconnect-with-backoff, id-mismatch → drop + `backend` envelope. `--broker-grants` argv (wasm.rs:77-81) and `RIZ_BROKER_RESOURCES` (`main.rs:1408-1418`, deleted) are replaced by `RIZ_BROKER_SOCK`/`RIZ_BROKER_TOKEN` minted per worker spawn (ProcessManager holds an optional `Arc<BrokerHandle>`; grantless functions get no env → local `denied` envelope, zero IPC). `src/broker/pg.rs` swaps `TokioPgBackend` (single lazy conn) for `SqlxPgBackend` over one shared `PgPool` per `[resources.pg.<name>]` (new keys `max_connections=5`, `acquire_timeout_ms=1000`; `statement_timeout` via after_connect; rustls TLS; READ ONLY tx; the exact text-param/`$N::type` + decode matrix preserved — `tests/broker_pg.rs` + `tests/pg_wire_mock` are the oracle). DSN resolution becomes daemon-startup fail-fast for grant-referenced resources.

**Tech Stack:** sqlx 0.8 (runtime-tokio, tls-rustls, postgres) — deny.toml/SBOM gate immediately after adding; deadpool-postgres over tokio-postgres is the pre-agreed fallback. getrandom for 32-byte tokens.

## Global Constraints
- Same gates; `--squash --admin` merge. All new src/ under Power of 10: bounded channels only (clippy denies unbounded), `try_acquire` never queue, supervised loops per SAFETY rule 2 (spawned task, await every iteration, bounded work, shutdown exit), no unwrap/indexing.
- The dispatcher seam (`src/broker/mod.rs` ordering + closed error set) survives verbatim; only backend construction + location change. Verb dispatch moves daemon-side (child forwards all verbs; `unknown_verb_envelope` relocates to the service).
- Token registry bounded: per function, at most 2×concurrency live tokens (mint evicts oldest — dead workers' tokens age out).
- e2e fleet configs gain NO live grant (daemon fail-fast would demand DSNs); the orders-wasm capstone ships as handler code (`persisted` field, denied→null, never fails the order) + commented riz.all.toml block + a mock-backed process-tree test.
