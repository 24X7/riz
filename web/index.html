// configuration

One file.
All your functions.

Define your entire function fleet in a single riz.toml. Routing, process management, and resource limits — automatic.

Point any HTTP client at Riz exactly as you would at API Gateway. Same format, different bill.

bun python — soon rust — soon node — soon websocket
riz.toml toml 
# one function = one process pool = N routes
# mirrors AWS Lambda + API Gateway v2

[function.api]
runtime     = "bun"
handler     = "src/api/index.handler"  # AWS file.export
timeout_ms  = 30000
concurrency = 10

# Implicit default: ANY /api
# Or explicit, with AWS-style path params + greedy proxy:
[[function.api.routes]]
path   = "/api/{id}"
method = "GET"

[[function.api.routes]]
path   = "/api/{proxy+}"
method = "ANY"
// MCP server, built in

Your APIs.
An LLM's hands.

Riz ships a spec-compliant MCP server at /_riz/mcp. Every function in your riz.toml automatically becomes a tool an LLM can invoke.

No SDKs, no wrappers, no second runtime. Drop your existing Lambda code in, point Claude or Cursor at the endpoint, and your APIs are agent-callable.

jsonrpc 2.0 mcp 2024-11-05 batched
claude mcp add riz bash 
# Point Claude / Cursor / any MCP client at your riz instance.
$ claude mcp add riz-local \
    --transport http \
    http://localhost:3000/_riz/mcp

# Now every function in riz.toml is an LLM tool.
# Claude can call them directly:

> tools/call api { id: "42" }
{
  statusCode: 200,
  body: { "id":"42","name":"Account 42" }
}

v0.1, shipping in public.

✓ Works now

  • AWS API Gateway v2 HTTP payload — exact aws_lambda_events types
  • Bun (TypeScript / JavaScript) handlers
  • Function-centric config: [function.<name>] + N routes per function
  • AWS path syntax: {id}, {proxy+}, $default
  • AWS handler syntax: handler = "index.handler"
  • Hot-swap deploys from S3 with in-flight request drain
  • riz.toml hot-reload on save
  • /_riz/health · /_riz/metrics · /_riz/registry
  • MCP server (spec 2024-11-05, JSON-RPC, batches, lifecycle)
  • Terminal dashboard: P50/P75/P90/P95/P99 over 5-min window
  • Datadog metrics emitter
  • WebSocket APIs ($connect / $disconnect / $default) + @connections management API at /_riz/connections/{id}

🚧 Coming
