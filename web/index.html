One file.
All your functions.

Define your entire function fleet in a single riz.toml. Routing, process management, and resource limits — automatic.

Point any HTTP client at Riz exactly as you would at API Gateway. Same format, different bill.

runtimes bun python rust
protocols http api v2 websocket
riz.toml toml 
# one function = one process pool = N routes
# mirrors AWS Lambda + API Gateway v2

[function.api]
runtime     = "bun"
handler     = "src/api/index.handler"  # AWS file.export
timeout_ms  = 30000
concurrency = 10

# Implicit default: ANY /api
# Or explicit, with AWS-style path params + greedy proxy:
[[function.api.routes]]
path   = "/api/{id}"
method = "GET"

[[function.api.routes]]
path   = "/api/{proxy+}"
method = "ANY"

Features.

Run AWS HTTP API v2 + WebSocket Lambda handlers in TypeScript, Python, or Rust on a single Rust binary. Hot-reload, hot-deploy, real percentiles, and a built-in MCP server that makes every function a Claude Code tool.

Features

v0.1 · in production

Function Runtimes

  • Bun (TypeScript / JavaScript) handlers
  • Python runtime adapter
  • Rust runtime adapter

Protocol Surface

  • AWS API Gateway v2 HTTP payload — exact aws_lambda_events types
  • WebSocket APIs ($connect / $disconnect / $default) + @connections management API at /_riz/connections/{id} — handlers in Bun, Python, and Rust
  • WebSocket connection list endpoint — GET /_riz/connections

Configuration & Routing

  • Function-centric config: [function.<name>] + N routes per function
  • AWS path syntax: {id}, {proxy+}, $default
  • AWS handler syntax: handler = "index.handler"

Deployment & Lifecycle

  • Hot-swap deploys from S3 with in-flight request drain
  • riz.toml hot-reload on save
  • Real Lambda context — getRemainingTimeInMillis, functionName, invokedFunctionArn, awsRequestId

Observability

  • /_riz/health · /_riz/metrics · /_riz/registry
  • Terminal dashboard: P50/P75/P90/P95/P99 over 5-min window
  • Datadog metrics emitter
  • MCP server — spec 2025-11-25 (also negotiates 2024-11-05 / 2025-03-26 / 2025-06-18). JSON-RPC 2.0, full lifecycle.

Agent & AI Integration

  • Claude Code & Cursor: every function auto-registers as an MCP tool at /_riz/mcp
  • Zero SDK code — point any MCP client at riz, your APIs become agent-callable
  • Bearer-token gating on /_riz/mcp for shared-secret access control

Developer CLI

  • riz init <template> — scaffold a working project (6 templates: TS/Python/Rust × HTTP/WebSocket)
  • riz run — boot the runtime with riz.toml + handler-source hot reload
  • riz validate / riz routes — config check + route table
  • riz deploy — S3 hot-swap deploy with auto health-check rollback

Security & Auth

  • Lambda authorizers — REQUEST + JWT (JWKS, TTL cache)
  • CORS auto-preflight — OPTIONS → 204, Access-Control-Allow-* headers
  • Bearer-token auth on /_riz/* (/_riz/health stays open)

On-Box Safety

  • On-box safety profile — always-on RLIMIT_CORE=0 + RLIMIT_NOFILE=4096 + RLIMIT_FSIZE=100MiB
  • Opt-in per-function caps — memory_mb (RLIMIT_AS), cpu_time_secs (RLIMIT_CPU), allowed_paths (Linux Landlock filesystem allowlist)

Roadmap

v0.2 and beyond

Capability-Sandboxed WASM

  • WebAssembly / WASI handlers wrapped in a host process — the same line-delimited JSON protocol as the other runtimes
  • Filesystem, network, and clock capabilities granted explicitly in riz.toml — no implicit host access
  • Sub-millisecond cold start, one .wasm binary across Linux / macOS / edge
  • The differentiator no Lambda emulator ships: safe execution of code you didn't write

Additional Runtimes

  • Node.js native runtime — for shops that won't ship Bun in prod
  • Go support via the existing static-binary protocol (thin riz-go-runtime module + templates; the runtime kernel is reused from Rust)
  • Java / JVM runtime adapter

Smarter MCP

  • Per-route MCP tool schemas — typed input shapes from path + query parameters
  • AI inspection tools — riz.tail_logs, riz.replay_request, riz.scaffold
  • OAuth 2.1 + RFC 8707 Resource Indicators (bearer-token path remains default)

Event Sources

  • Non-HTTP event sources (SQS, SNS, S3, EventBridge) — v0.2

Distributed Tracing

  • OpenTelemetry exporter with W3C Trace Context propagation (X-Ray headers come free)
